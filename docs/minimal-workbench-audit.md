# 极简工作台审计与改进计划

目标：让 TTS More 像一个配音生产工作台，而不是模型配置控制台。默认路径要短，异常信息要清楚，高级能力要留在需要时再展开。

这份文档是当前阶段的产品简化账本。它记录已经关闭的问题、仍然值得做的最小下一步，以及验证方式。不要把旧阶段描述当成最新事实；代码、测试和浏览器状态始终优先。

## 工作台契约

```mermaid
flowchart LR
  A["剧本"] --> B["提取台词"]
  B --> C["角色音色"]
  C --> D["TTS 接入"]
  D --> E["生成台词"]
  E --> F["行内试听历史"]
```

三栏只承担三件事：

- 左侧：当前剧本、原文和提取入口。
- 中间：台词列表、筛选、选行、批量生成和历史试听。
- 右侧：当前行的音色、参考资源、生成文本和生成本行。

顶部入口使用任务名：

- `角色`：给当前剧本角色选择或保存常用音色。
- `队列`：有生成任务时查看分发状态和最近任务。
- `解析`：配置剧本解析用 LLM。
- `接入`：粘贴并检测 TTS Gradio endpoint。

## 仍待处理的问题

| 优先级 | 问题 | 用户看到的症状 | 最小改动 |
| --- | --- | --- | --- |
| P0 | Inspector 仍要求用户先理解 GPT / Index / Cosy / API | 生成前像在选择模型控制台，而不是确认当前音色 | 默认显示当前音色、当前服务、资源缺口和生成；切换方式放进更多设置 |
| P0 | 顶部 `队列` 在空队列时仍可能成为重复入口 | 普通状态下多一个无任务弹窗 | 无任务时隐藏或降级；生成中从进度入口打开详情 |

| P1 | 角色与音色仍有全局维护心智成本 | 给当前角色选音色时会看到身份、标签、扫描、绑定表单 | 默认只显示当前项目音色和常用音色；维护折叠 |
| P1 | 台词行卡在普通状态仍有偏诊断的 chip | 未生成台词也像排障页面 | 常态只保留角色、文本、状态、播放/生成；异常才显示诊断 |
| P2 | 移动端台词行仍可能依赖宽表格 | 390px 视口需要横向滚动 | 移动端行卡单列化 |

## 文案规则

- 正常状态少说话，只给当前可做动作。
- 异常状态说清楚发生了什么，以及下一步该做什么。
- UI 使用用户任务语言：`生成本行`、`检测并保存`、`保存为常用音色`。
- 文档可以解释内部实体，但主路径不主动暴露 `cluster key`、`profile`、`binding_id`、旧 repo path。
- 给 Agent 的说明要短、稳定、可验证；不要让 LLM 进入过深的实现推理。
