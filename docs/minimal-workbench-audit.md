# 极简工作台审计与改进计划

目标：让 TTS More 像一个配音生产工作台，而不是模型配置控制台。默认路径要短，异常信息要清楚，高级能力要留在需要时再展开。

设计原则：

- 一屏只回答当前任务，不展示所有系统能力。
- 能合并成一个动作的，不拆成两个同级按钮。
- 能用用户语言命名的，不用内部实体命名。
- 正常状态少说话；异常状态给下一步。
- 文档先讲怎么完成工作，再讲为什么这样实现。

## 审计范围

- 前端主工作台：`frontend/src/App.tsx`、`frontend/src/App.css`、`frontend/src/i18n.ts`。
- 工作台辅助逻辑：`frontend/src/lib/workbenchView.ts`、`frontend/src/lib/workstation.ts`。
- 人类/Agent 文档：`README.md`、`docs/open-source-tts-services.md`、`docs/release-governance.md`、`frontend/design.md`、`docs/agent-prompts/gpt-sovits-fork-enhancement.md`。

## 当前最短流程

```mermaid
flowchart LR
  A["打开或新建剧本"] --> B["提取台词"]
  B --> C["匹配角色音色"]
  C --> D["检测并保存 TTS endpoint"]
  D --> E["选择台词并生成"]
  E --> F["试听历史版本"]
```

## 已确认问题

| 优先级 | 问题 | 影响 | 改进方向 |
| --- | --- | --- | --- |
| P0 | 文档写 `服务与资源`，实际顶部是 `角色库 / 资源队列 / LLM API / TTS 配置` | 用户和 Agent 会按不同名字找入口 | 统一为任务名：`角色 / 队列 / 解析 / 接入` |
| P0 | 角色库弹窗同时承担模型映射、角色编辑、候选扫描、项目绑定 | “给角色选音色”被内部概念淹没 | 主路径改成“剧本角色 -> 推荐模型 -> 用于本剧本”，全局编辑折叠 |
| P1 | 右侧检查器默认展示方法说明、三联配置、声音资源、诊断 | “生成本行”被配置噪音包围 | 默认展示必要摘要和生成按钮；配置/诊断按需展开 |
| P1 | TTS 接入把 `检测配置` 和 `保存端点` 放成两个主动作 | 容易跳过检测，保存不可用端点 | 合并主动作：`检测并保存` |
| P1 | 剧本控制台同时有管理、编辑、预览、提取 | 新用户不知道第一步 | 空态只引导新建/导入；已有剧本突出提取台词 |
| P2 | UI 文案残留“历史项目路径”“历史本机项目”等兼容词 | 增加学习成本，误导 Agent 走旧路径 | 兼容说明放文档，不放主 UI |
| P2 | CSS 多套 card/chip/meter 体系重复 | 后续改动容易局部发散 | 合并状态 chip、目录 row、配置 card 的样式语义 |
| P2 | 小屏仍有宽表格遗留 | 移动端核心路径需要横向滚动 | 行卡真正单列化，次级 chip 收进展开区 |

## 本轮已落地

- 顶部入口文案收敛为 `角色 / 队列 / 解析 / 接入`，英文同步为 `Roles / Queue / Parser / Setup`。
- 开源 TTS 接入新增主动作 `检测并保存`，避免用户或 Agent 跳过检测。
- 右侧生成方式 tab 去掉默认可见解释文，解释转入 `title` 和 `aria-label`。
- 诊断按钮在正常状态只显示状态；只有存在诊断时显示展开/收起动作。
- README 改为先给最短操作路径，再链接详细文档。
- 发布治理补上 `Project/`、`data/parser_providers.json`、`.omc/`、`.omo/` 的不可提交边界。

## 后续任务分解

### 阶段 1：入口和文案一致

- 搜索 `服务与资源`、`配置管理`、`历史项目路径`、`历史本机项目` 等旧词。
- 保留仍被代码使用的兼容 key，但避免在主路径显示。
- README、接入文档、发布治理使用同一套入口名。
- 验证：i18n 测试覆盖核心入口名。

### 阶段 2：右侧检查器默认收敛

- 新增“当前将使用”摘要：生成方式、音色、服务、参考来源。
- 默认折叠 profile/binding/service 三联配置。
- 保留错误和缺配置状态自动展开。
- 验证：补充 `workbenchView` 测试，覆盖正常态/缺配置态/历史版本态。

### 阶段 3：角色库主路径重排

- 默认区域只服务“当前剧本角色绑定”。
- `写入角色库` 改名为 `保存为常用角色`。
- 全局角色身份、标签、别名、备注编辑移入折叠区域。
- 扫描候选和模型目录只在用户刷新或搜索时强调。
- 验证：浏览器烟测 `角色 -> 选择模型 -> 用于本剧本`。

### 阶段 4：剧本控制台空态简化

- 无剧本时只突出 `新建/导入剧本`。
- 有剧本时突出 `提取台词`，管理入口保持次级。
- 版本风险确认文案继续说明“不覆盖旧台词和音频”。
- 验证：空项目和已有项目两种状态截图检查。

### 阶段 5：文档重写收束

- `frontend/design.md` 改成 TTS More 设计基线，Geist token 放附录。
- `docs/agent-prompts/gpt-sovits-fork-enhancement.md` 删除旧行号和大段补丁代码，改成可复用任务契约。
- README 只保留新人入口和文档索引，高级机制放专题文档。
- 验证：Agent 首读 README 能在 2 分钟内找到启动、接入、生成和发布边界。

## 暂缓事项

- 大规模删除 `servicePanelSection === "overview"` / `"tts"` 旧分支：需要先确认是否仍作为隐藏管理入口使用。
- 前端拆包：Vite 当前只有 chunk warning，功能和测试通过；应单独评估动态导入边界。
- Starlette/httpx2 迁移：属于依赖兼容主题，应另开小任务验证 FastAPI/TestClient 影响。
